# /draft-post

Bundle existing atomic notes into an editorial post. The LLM (Sonnet 4.6) is constrained to cite each section against your sources via wikilinks, so the post stays anchored in actual material rather than drifting.

## What it does

1. Resolve source slugs from `--notes` against `.data/index.db`. Missing slug → typed error before any LLM call.
2. LLM synthesize (Sonnet 4.6): produce `{ title, lede, sections: [{heading, body}], conclusion }`. Each section body MUST cite at least one source via `[[<slug>|surface]]`.
3. Validate via Zod + citation check. On failure, retry once with an explicit reminder. On second failure, exit 1 — no file written.
4. Compose final markdown:
   - Frontmatter from `--title`, today's date, and the union of source tags (capped at 5).
   - Body: lede → sections → `## Conclusion` → `## Sources` (auto-generated, listing each source as `[[<slug>|<title>]]`).
5. Atomic write to `content/posts/<slug>.md` with `draft: true`.

The Sources block is generated by the script, not the LLM, so it is always complete and ordered to match `--notes`.

## Run

```bash
bun run scripts/draft-post.ts \
  --title "Why RLHF is brittle" \
  --notes 01-deep-rl-from-human-preferences,09-ppo,11-rlpr

# flags
--force      overwrite an existing target post
--dry-run    skip the LLM call; print prompt size + planned target
```

## Prerequisites

- `.data/index.db` must exist with the source slugs already ingested. Run `bun run build-index --skip-embed` (or full `bun run refresh`) first.
- Source slugs are the file IDs (= filename without extension) under `content/notes/` or `content/posts/`.

## Cost

Sonnet 4.6 with 8K output budget. Per post: ~$0.05-0.20 depending on source body length (each source body capped at 8K chars). Posts are infrequent — a few per week — so the synthesis-quality premium over Haiku is worth it.

Run `--dry-run` first to confirm source loading + size before paying for synthesis.

## Style guarantees

The system prompt explicitly bans common AI-slop phrases ("in conclusion", "furthermore", "delve into", "tapestry", "underscore", etc.) and asks for concise, editorial voice. Quality of long-form synthesis still varies — review the draft before flipping `draft: false`.

## Idempotency

Refuses to overwrite an existing target unless `--force` is set. Atomic write via `.tmp` + rename keeps a mid-write crash from leaving half-files.
